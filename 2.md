# My Journey into Backend Development

Transitioning from writing strictly front-end code with HTML, CSS, and styling libraries into server-side logic has been an eye-opening experience.

## Why Backend?

While front-end development is incredibly visually rewarding, a lot of the heavy lifting for modern applications relies on APIs, databases, and secure servers. To build truly dynamic AI-driven tools, I needed a way to securely interact with those external APIs without exposing my keys.

## Getting Started with Node.js

Node.js allowed me to take the JavaScript I already knew from the browser and run it natively on a server. It was fascinating to write a simple application that listened for HTTP requests and responded with data.

```javascript
const express = require('express');
const app = express();

app.get('/api/status', (req, res) => {
    res.json({ status: "Server is running perfectly!" });
});

app.listen(3000, () => console.log('Listening on port 3000'));
```

## Connecting Systems

What surprised me the most was how backend development connects everything. Building a bridge between a frontend form, a backend endpoint, and an AI service like Gemini feels like giving the application a real "brain".

I'm excited to continue this journey, exploring Express middleware, database integration, and more advanced security practices!
